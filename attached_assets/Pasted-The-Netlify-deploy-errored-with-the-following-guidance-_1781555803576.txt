  - [line 74](#L74) is the actual error: "Error: PORT environment variable is required but was not provided."
  - [lines 69вЂ“71](#L69-L71) show the build command that failed (pnpm ... run build).
  - The config path referenced in the logs is artifacts/mockup-sandbox/vite.config.ts (from [line 75](#L75)). You can inspect that file in the repo: https://github.com/rakju-ui/MOG-MONKEY/blob/main/artifacts/mockup-sandbox/vite.config.ts

Diagnosis
- Error type: runtime exception thrown while loading Vite config (vite.config.ts).
- Cause: vite.config.ts requires a PORT environment variable and throws when it is not present. During Netlify builds the PORT env var is not set, so vite fails to load the config and the build exits with code 1 (see [line 74](#L74)).

Solution (two options вЂ” pick one or do both)

1) Quick fix (set PORT in Netlify build environment)
- In the Netlify site settings, add an environment variable named PORT with a sensible default (e.g. 5173, ViteвЂ™s default). Steps:
  - Netlify UI: Site > Settings > Build & deploy > Environment > Environment variables > Add variable
    - Key: PORT
    - Value: 5173
  - Or add to netlify.toml in your repo (commit this file) to set the build-time env var:
```toml
[build.environment]
  PORT = "5173"
```
- Re-run the Netlify deploy. This will satisfy the required variable and allow vite.config.ts to load.

2) More robust fix (change vite.config.ts to provide a default instead of throwing)
- Edit artifacts/mockup-sandbox/vite.config.ts to avoid requiring PORT at load-time. Provide a fallback and only use PORT where appropriate (or avoid throwing). Example change to make the config safe for build environments:

```ts
// artifacts/mockup-sandbox/vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig(({ command }) => {
  // Use provided PORT or fallback to 5173
  const port = process.env.PORT ? Number(process.env.PORT) : 5173;

  // If you previously threw when PORT was missing, remove that throw.
  // Return your existing config but use `port` where needed.
  return {
    server: {
      port,
    },
    // ...rest of config
  };
});
```

- Commit and push the change, then re-run the Netlify build.

Notes
- Recommended: implement the code change so local dev and CI builds do not rely on an externally-set env var, and still set PORT in Netlify if your app expects a specific port.
- After applying either fix, trigger a redeploy. The build should progress past the vite config load step (previously failing at [line 72](#L72) / [line 74](#L74)).
